# 第 02 章 算力作为生产要素：定义、计量与三重身份

## 本章概览

第 1 章把一张 H100 SXM5 拆到物料清单（BOM）粒度——单卡物理成本 \$3,320、渠道售价 \$25,000-30,000、11 个环节跨 4 个国家、HBM 占 BOM 41% 超过 GPU 裸片。那张表回答了算力的物理面是什么，没回答算力在经济学里是什么。

回答这件事比想象的难。同一件物理事实——一台机柜每秒做了多少次浮点运算——在四个角色眼里是四套不可通约的数字：

- **AI 工程师**：nvidia-smi 里那一行 70% 利用率，PR 评论里 "this model was trained on 128× H100 for 36 hours" 的训练日志
- **数据中心运维**：1.2GW 总功率、PUE 1.15、单机柜 70kW
- **CFO**：季度资本支出 \$28B、加速器折旧期 5 年、融资租赁余额 \$260B
- **模型公司销售**：GPT-4.1 nano \$0.10/\$0.40 per 1M tokens、H100 现货价 \$2-3.5/hour、预留实例（reserved instance）12 个月 -40%

跨部门开会时，四套语言对不上。

更难的是，算力在企业财务里同时是三种不同的会计对象：

- 对微软、Meta、xAI 这些自建超大规模云厂，算力是**资本品**。进固定资产（PP&E）按 5-6 年直线折旧，每季度的折旧摊销（D&A）从经营利润里扣掉。
- 对 CoreWeave 这样的转租玩家，算力既是**资产也是抵押品**。单卡 H100 用作优先级有担保定期贷款（senior secured term loan）的抵押物，借出来的钱再去买更多卡。
- 对 OpenAI 这样的模型公司，算力是**运营支出加照付不议（take-or-pay）合同**。签的是 5-7 年 \$300B 量级的承诺，履约义务跟租户对房东的关系一样硬。
- 对终端调 API 的应用开发者，算力是**按 token 计价的现货商品**。价格随模型版本月度变动，像期货合约一样有合约方差。

> 术语速查：超大规模云厂（hyperscaler）= 亚马逊云、Azure、GCP、Meta、Oracle 等运营 10 万以上服务器的超大规模云。PUE 即数据中心总能耗除以 IT 设备能耗。资本支出（CapEx）即购置长期资产的现金流出。融资租赁（finance lease）即会计上等同于借钱买资产的租赁。折旧摊销（D&A）即 Depreciation & Amortization。

这套语言系统需要一次完整翻译。三件事要钉住：FLOP / GPU-hour / token / MW / 资本支出五个量纲的换算速查；算力作为生产要素、资产、商品的三重经济身份；Solow / Jevons / Acemoglu / Bresnahan-Trajtenberg / Minsky / Soros 六个理论锚点的一张矩阵。

涉及具体公司（微软、Meta、Google、亚马逊、CoreWeave、英伟达、OpenAI）的折旧政策、财务披露、估值含义，这里只给方法论与基础数据；Burry 反情景与超大规模云厂长折旧期的争议，先把两条证据线摆出来。

下面举三个例子。

**看到超大规模云厂财报上 \$80B 资本支出这一行**，能在脑子里翻译成：约 250 万张 H100 等价物 × \$32K 整机成本，每年 \$13-16B 折旧。这里假设 100% 投在 GPU 上；实际资本支出还含数据中心建设、网络、电力，按 §1.2 综合 \$50M/MW 折约 1.6 GW 新增容量。

**看到 OpenAI 与 CoreWeave 的 \$22.4B / 5 年合同**，能翻译成：约 75K H100 长租，6 GW-hour 累计算力，1 倍量级的合同储备对应自身年化营收（每年 \$4.5B 对应 OpenAI 2025 年化经常性收入 \$22-25B）。

**看到 DeepSeek-R1 首发价 \$0.07 / 1M token**（2025-01-20），能翻译成：输入价较 GPT-4 跌 400 倍，Jevons 反弹 5,000%，英伟达单日蒸发 \$589B。

## 1. 五套单位之间的换算速查表

算力的计量单位不是一个量纲，是五个不可通约的量纲在不同行业被混用。把它们摆在一张表里之前，先把每一套单位的物理含义和默认陷阱说清楚。

### 1.1 五个量纲与各自的原生行业

**FLOP / FLOPS**：单次浮点运算与每秒浮点运算次数。FLOPS 是工程师与硬件评测的母语。

一张 H100 SXM5 的 Tensor Core 算力按精度档分级：FP8 dense 3,958 TFLOPS（含 2:4 sparsity 7,916 TFLOPS），FP16 / BF16 dense 1,979 TFLOPS（含 sparsity 3,958 TFLOPS），TF32 dense 989 TFLOPS（含 sparsity 1,979 TFLOPS），FP32 67 TFLOPS。

前沿模型训练量按 FLOP 计。GPT-4 训练量约 2e25 FLOP，2030 年单次训练可达 2e29 FLOP。

**GPU-hour**：单卡运行 1 小时的工作量，云算力市场的主要计量单位。Silicon Data H100 Rental Index 2026-Q1 给出的 H100 长期合约价 \$2.35/hour。预留实例与现货价分化大，§1.3 单独讨论口径陷阱。

**token**：语言模型输入或输出的最小语义单位，约等于 0.7 个英文单词或 1.5 个汉字。终端 API 定价按 per million tokens 计。

- GPT-4（2023-03）：\$30 输入 / \$60 输出 per 1M tokens
- GPT-4.1 nano（2025-04）：\$0.10 / \$0.40 per 1M tokens

三年里同等能力档的输入价跌了 300 倍。DeepSeek-R1（2025-01-20 发布）首发价 \$0.07 per 1M tokens 输入，与同期 OpenAI 同档模型差 20-50 倍。

**MW**：数据中心电力规模的主要单位。一台 8 卡 HGX H100 整机额定 6-8kW，1MW 数据中心容纳 125-160 台整机。

Crusoe Abilene 旗舰项目园区规划总容量 1.2GW，2025-03 扩容公告，首批两栋建筑约 200MW。

超大规模云厂锁电、PJM 容量拍卖、PPA 合同都按 MW 计价。

> PJM 是美国最大区域电力市场运营商，覆盖宾州、新泽西、马里兰等 13 州加华盛顿特区。PPA 即 Power Purchase Agreement 购电协议。

**资本支出**：财务报表语言。

- 微软 FY25 资本支出 \$64.6B（截至 2025-06-30）
- Meta FY25 资本支出约 \$72B
- 亚马逊 FY25 资本支出 \$131.8B（截至 2025-12-31）

注意三家财年起止不同：微软 FY25 截至 2025-06-30，亚马逊 / Meta FY25 截至 2025-12-31。后续章节按时点对齐做横向比较时会重新折算口径。

资本支出不分芯片 / 服务器 / 建筑 / 电力细类，公司一般只在管理层讨论与分析（MD&A）里说含 AI 基础设施。市场拿到的是合并资本支出数字。

### 1.2 五维换算速查表

把五个量纲放在一张表里——纵向是源单位，横向是目标单位，每个格子写换算系数和默认假设。

| 源 → 目标 | FLOP | GPU-hour | token | MW | 资本支出 (USD) |
|---|---|---|---|---|---|
| **1 FLOP** | 1 | 7e19 FP8 FLOP ÷ 3,958 TFLOPS ÷ 3,600 s ≈ 1 GPU-hour 的 1.42e16 倍 | n.a.（FLOP→token 依模型而定，按 ~6 × P × T 推算训练，按 ~2 × P × T 推算推理；P=参数量，T=tokens 数量） | n.a. | n.a. |
| **1 GPU-hour (H100)** | 3,958 TFLOPS × 3,600 s × 利用率 ≈ 1.42e19 FP8 FLOP（理论上限，含 sparsity）；按 50% 利用率折算 ~7e18 | 1 | 推理 H100 单卡 ~3,000-5,000 tokens/s（业内估算，依模型 + 批量大小 + KV 缓存设计而变） × 3,600 s = 1e7-1.8e7 tokens/小时 | 单卡满载 0.7kW，按 PUE 1.15 折 0.0008MW | 单卡整机 \$32,000、按 5 年直线 + 利用率 85%：每 GPU-hour 折旧成本 ~\$0.86；运营成本（电 + 冷却 + 人员 + 网络）业内估算 \$0.3-0.5/h，总成本 ~\$1.2-1.4/h |
| **1M tokens（输出）** | 推理 ~2 × P × 1M，P=70B 即 1.4e17 FLOP，P=405B 即 8.1e17 FLOP | 70B 模型 H100 单卡 ~80 s（业内估算），即 0.022 GPU-hour；405B 模型 ~470 s，即 0.13 GPU-hour | 1 | 折算 MW 无意义（瞬时） | 折旧 + 运营摊销 ~\$0.026-0.18（业内估算），市场零售价 \$0.4-15（依模型档位）|
| **1 MW (年化)** | 1MW × 8,760 h × 1,250 GPU/MW（按整机分摊）× 1.42e19 = 1.55e26 FP8 FLOP / 年（理论上限） | 1,250 × 8,760 = 1.1e7 GPU-hour / 年 | 1.1e7 × 1.4e7 tokens = 1.5e14 tokens / 年（理论上限） | 1 | 数据中心资本支出 \$10-13M/MW（CBRE / JLL 2024-2025 基准）+ GPU 整机 \$40M/MW（按 1,250 卡 × \$32,000）= 总 ~\$50M/MW |
| **1B USD 资本支出** | n.a. | \$40M / GPU = 25,000 卡 × 8,760 × 85% = 1.86e8 GPU-hour 的 5 年累计输出（按 80% 注 GPU、20% 注电 + 楼宇）| 衍生计算 | 20MW（按 \$50M/MW 综合）| 1 |

> 来源：换算系数综合英伟达 H100 产品规格页（FP8/FP16 FLOPS）、Silicon Data H100 Rental Index 2026-Q1、CBRE / JLL 2024-2025 数据中心建设成本基准、Epoch AI 2024-2026 推理 FLOP / token 经验公式、Silicon Analysts AI Server Cost 数据库 2025（单整机 \$32K per GPU 摊销口径）。**所有利用率假设属业内估算**——实际利用率随工作负载、客户类型、合约形态浮动 30-95%。本表用作单位之间数量级翻译，不可逐格作为投资测算输入。

读这张表的方式不是逐格对照1 FLOP 等于多少 GPU-hour——这种换算大多没有物理意义，因为转换效率随模型 / 工作负载 / 利用率变动一个数量级以上。这张表的用法是：拿到一个数字，先看它原生在哪一列，再决定它能不能跨列翻译。

举三个典型场景。

### 场景一：训练 1 个 GPT-4 量级的模型要花多少钱

训练量按 FLOP 计。GPT-4 业内估算约 2e25 FP8 FLOP。按 H100 FP8 算力 3,958 TFLOPS、平均利用率 35-40%（前沿训练利用率受同步开销与显存约束影响），单卡每小时实算约 5e18 FP8 FLOP。

2e25 ÷ 5e18 = 4e6 GPU-hour。按 GPU-hour 业内综合成本 \$2/h（含电力、折旧、网络），训练 GPT-4 约 \$8M 纯算力成本。

考虑失败重跑、实验、数据预处理，全程业内估算 \$30-100M。Epoch AI 公开估算 GPT-4 \$78M。口径区间宽，依失败次数、利用率、数据准备假设变动。

这条换算链有 4 层假设，每层 ±30%，最终区间 ±数倍。任何把这一类数字写成点估计的报道，都漏掉了口径方差。

### 场景二：1MW 数据中心一年能产多少 token 营收

按 1MW = 1,250 卡 × 8,760h × 利用率 85% × 推理速度（70B 模型业内估算约 3,500 tokens/s）= 2.9e13 tokens / 年。

按 GPT-4o-mini 档位定价 \$0.6/M tokens 输出卖，理论营收上限约 \$17.4M / MW / 年。

实际营收远低于这个数。大部分推理是免费层、缓存命中或内部用，付费 token 占比业内估算 10-20%。

1MW 折 \$1.7-3.5M 净 token 营收——这是 GPU 云能不能赚钱的最朴素下限。

### 场景三：单卡每小时折旧成本怎么算

单卡整机 \$32,000 ÷ 5 年 ÷ 8,760h × 1/0.85（利用率）= \$0.86/h。这是超大规模云厂按 5 年折旧的最佳情况。

Michael Burry（《大空头》原型）在 2025-11 公开主张超大规模云厂 GPU 真实折旧应为 2-3 年。如果按 3 年折旧，单卡每小时折旧 \$1.43。

这件事看起来只是会计选择，但直接决定超大规模云厂单卡是否盈利。详情见 §3。

### 1.3 三类口径陷阱

读 AI 算力数据时，三类口径陷阱让同一指标 ±2 倍成为常态。每一类都给一个产业里反复出现的真实案例。

#### 陷阱一：浮点位宽与稀疏化的标称差异

H100 SXM5 的 Tensor Core 算力按精度档分级，英伟达官方规格 dense 与 sparsity 同时列示：

| 精度 | Dense | Sparsity |
|---|---:|---:|
| FP32 | 67 TFLOPS | 不适用 |
| TF32 | 989 TFLOPS | 1,979 TFLOPS |
| BF16 / FP16 | 1,979 TFLOPS | 3,958 TFLOPS |
| FP8 | 3,958 TFLOPS | 7,916 TFLOPS |
| INT8 | 3,958 TOPS | 7,916 TOPS |

先解释这两列的含义。**Dense（稠密）** 指矩阵里所有权重都参与计算，是平时写代码跑出来的真实算力。**Sparsity（稀疏）** 指 Tensor Core 硬件支持的 2:4 结构化稀疏——每 4 个连续权重强制保留 2 个、裁掉 2 个，硬件直接跳过零值的乘加，吞吐翻一倍。所以表里 Sparsity 列永远是 Dense 的 2 倍。

> FP4 是 Blackwell 代新增的 4-bit 浮点；TF32 是英伟达自定义的 19-bit 浮点，FP32 的截断版，Tensor Core 专用。

两个细节卡住大量营销与卖方数字。

**第一，同一张卡换口径，报数可以差 4 倍。** 规则是：精度位宽减半算力翻倍（FP16 → FP8 → FP4），开 sparsity 再翻倍。所以 H100 FP8 sparsity 的 7,916 TFLOPS 与 FP16 dense 的 1,979 TFLOPS 之间差 4 倍——同一张卡，只是口径变了。

英伟达营销材料常用 sparsity 加速档作头条数字。但训练实战大多用 BF16 / FP8 mixed precision，sparsity 加速对前沿模型几乎不开——开了精度会掉。算真实算力按 BF16 dense 1,979 TFLOPS 起步更靠谱。

看到算力数字先问两句：精度档是 FP 几？sparsity 开没开？

**第二，跨代比较按同一档算，差距能从 1.2 倍跳到 5 倍。** H100 对 B200 在同档 dense FP8 单卡上跨代比只有 1.1-1.2 倍。换成 FP8 含 sparsity + NVL72 整柜（叠加 NVLink 5 与 HBM3e 带宽提升），业内常见报 1:2.5。再换到跨档对照——B200 FP4 对 H100 FP8——会跳到 5 倍。

Blackwell 是 Hopper 性能的 5 倍 这种话术通常默指 FP4 对 FP8 的跨档比对，不是同档跨代。

精度档、sparsity、单卡对整柜，三层口径叠起来让跨代加速比从 1.2 倍跳到 5 倍。

#### 陷阱二：GPU-hour 含不含网络、电、折旧

Silicon Data 公开的 H100 长期合约价 \$2.35/hour 是超大规模云厂 GPU 单卡每小时租金。这个数字在不同口径下含义不同：

- **仅算力（compute-only）**：只算 GPU 单卡，不含 CPU、内存、本地 SSD、网络。主要出现在 GPU 现货交易平台（Vast.ai、SF Compute），单价低，但客户要自己处理数据加载、检查点（checkpoint）上传。
- **裸金属（bare-metal）**：含整机（CPU、内存、本地存储），不含 InfiniBand 网络。客户拿到的是一台机柜的算力，能本地训小模型，但跨机柜跑大模型训练效率掉。
- **NVL36 / NVL72 整柜**：含 NVLink Switch、InfiniBand 内部互连、整机柜液冷。整柜算力加互联完整。CoreWeave、Lambda 的超大规模云厂长合约多按这种口径报。
- **托管集群（managed cluster）**：除了硬件还含 Slurm 或 Kubernetes 调度、监控、自动检查点、故障切换。超大规模云厂公有云上的口径，价格通常是裸金属的 1.5-2 倍。

> InfiniBand 是英伟达高带宽集群互连标准，单端口 400Gb/s。NVLink Switch 是英伟达自研的 GPU-GPU 高带宽互连协议，第 4 代单向 450 GB/s。Slurm 是高性能计算作业调度。

同一张 H100 在四套口径下 GPU-hour 价格可以从 \$1.8 跳到 \$4.5——区间因子 2.5 倍。看 GPU 云财报或行业研报时，先问这是哪一档口径。

CoreWeave 在招股说明书（S-1）里明确说自己卖的是 NVL36 加托管集群，毛利率 60%+；vast.ai 卖的是仅算力，毛利率 15-25%。两家放在一起比 H100 租金是错位对照。

#### 陷阱三：token 计费口径的方差

OpenAI、Anthropic、Google 的 API 定价表上有几个细则容易被忽略，但对实际成本估算影响巨大：

- **提示词缓存（prompt caching）**：重复的前缀部分按缓存输入计费，单价是原输入价的 10-25%（OpenAI 50%，Anthropic 10%）。对长上下文加重复模板的应用（如客服机器人），实际单 token 成本可以降到标价的 30-50%。
- **批处理 API（Batch API）**：单价为实时 API 的 50%。要求 24 小时内返回，适合后台数据处理，不适合实时交互。
- **上下文长度定价**：部分模型对 128K+ tokens 长上下文单价加价 2-4 倍。Gemini 1.5 Pro 在 128K+ 上下文上单价跳档。
- **推理 token（reasoning tokens）**：o1、o3、Claude Sonnet 4.5 思考模式的思考 token 按输出计费但用户看不到。一次查询可能产生几千个推理 token 加几百个可见输出 token，实际单次查询成本远高于可见输出 × 单 token 价。

这四个变量让 GPT-4 / Claude / Gemini 单价对比成为口径战场。一张漂亮的 Claude Sonnet 4.5 比 GPT-4o 便宜 30%对照表，常常只比可见输出的单价，不计推理 token、不计缓存折扣、不计批处理折扣。

报 token 单价前要说清楚口径。

### 1.4 工程师、投资者、财务的四种数字直觉

把同一台机柜的算力放在四类读者面前，他们看到的数字直觉不同。这种差异是各自工作语境的合理选择。但跨界沟通时如果不互译，开会十分钟说不到一起。

举一个具体例子：CoreWeave 2025 Q3 8-K 披露单季营收 \$1,364.7M。这个数字翻译成四种语言：

- **工程师视角**：CoreWeave Q3 平均部署活跃 GPU 量约 220K 卡。按 \$1.36B ÷ 90 天 ÷ 24 小时 ÷ \$2.85 加权平均小时价折算。这是一个能跑 32 个 Llama 4 Maverick 集群的规模。
- **运维视角**：220K 卡 × 0.7kW × PUE 1.15 = 177MW 总功耗。相当于 1.5 个 1GW 级数据中心的 GPU 部分，或者 30 个传统 enterprise 5MW 级数据中心。
- **财务视角**：\$1.36B 营收每季，年化 \$5.5B。按 60% Adjusted EBITDA margin = \$3.3B EBITDA。按 EV/EBITDA 12-15 倍估值 = \$40-50B 市值（不含杠杆调整）。
- **市场视角**：\$1.36B ÷ 220K 卡 ÷ 91 天 ÷ 24 小时 = \$2.85 / 卡-小时综合定价。这是超大规模云厂长合约加中小客户短租加权平均。

四种数字直觉描述同一物理对象，但在跨部门会议上很难无缝拼接。速查表与三类口径陷阱是后续读 GPU 云、超大规模云厂、模型公司财务数据时的对齐基准。

## 2. 三重身份之一：作为生产要素的 K_compute

把一张 H100 拿在手里掂量，工程师看到的是 GPU；CFO 看到的是固定资产上 \$32K 的资本品；经济学家看到的是 K——Cobb-Douglas 生产函数里那个进入生产函数的资本要素。三个视角看的是同一个物，但用来回答的问题不一样。

### 2.1 Cobb-Douglas 与 K 的分裂

宏观经济学最古老的生产函数是 Cobb-Douglas 形式：

```
Y = A · K^α · L^β
```

Y 是产出，A 是技术参数（也叫 Solow 残差或 TFP 全要素生产率），K 是资本，L 是劳动，α + β = 1 是规模报酬不变的约束。

Robert Solow 1957 年在《Technical Change and the Aggregate Production Function》里用这个框架做增长会计——把 1909-1949 美国人均产出年化增长率 1.5% 拆开，发现 87.5% 来自 A（技术、残差），只有 12.5% 来自资本积累。这个结果被反复引用为经济增长主要来自技术，不是资本堆积。

把算力放进这个框架，麻烦从 K 是单一变量开始。AI 算力跟传统资本品（厂房、机器、卡车）的差别有三件：

**一**，算力的技术含量远高于传统资本。一台 1980 年代日立的高炉跟一台 2020 年代日立的高炉差异有限。一张 2026 年的 B200 与 2024 年的 H100 跨代算力差按口径不同在 1.2-2.5 倍之间。同一美元资本投入带来的算力产出在跨代时跳变。

**二**，算力的折旧速度远快于传统资本。机器折旧期典型 10-20 年，GPU 在超大规模云厂按 5-6 年。算力作为资本，年化折旧率 17-20%；传统机器年化折旧率 5-10%。这件事让算力在生产函数里的有效存量远小于账面存量。详见 §3。

**三**，算力的产出弹性 γ 没有公认值。劳动份额 β 长期约 0.6-0.7，资本份额 α 约 0.3-0.4，这是经济学教科书共识。算力作为单独要素的产出弹性 γ 在学术界没有公认估算，原因有三：

- 算力作为生产要素历史太短，2010 年代深度学习兴起后才开始进入主流
- 算力分布高度不均，少数公司占大头
- 算力进入企业生产函数的渠道复杂——自建、租赁、调 API

把上述三件事压进生产函数，更细化的形式是：

```
Y = A · K^α · L^β · C^γ
```

C 是 K_compute，γ 是算力的产出弹性。问题：γ 多大？

### 2.2 γ 的三套估算

作者推演三套估算路径，对应低 / 中 / 高三种假设。

#### 路径一：从要素份额倒推（γ ≈ 0.02-0.04）

经济学里一个常见做法是按要素的收入份额等同于其产出弹性。这在完美竞争加边际产品定价的假设下成立。

2025 年全球 AI 算力相关支出（资本支出加云租赁加 token 消费）业内估算 \$400-500B。同年美国 GDP \$29T。算力支出占 GDP 约 1.5%。

按要素份额等于产出弹性的简化推论，γ 约 0.015-0.02。

但要素份额倒推法在算力上有两个缺陷：

- 算力进入生产函数有乘数效应——同样的劳动加同样的资本，配上算力后产出大幅提升（软件工程师配 Copilot）
- 算力支出在 GDP 里被部分重复计入——超大规模云厂资本支出进 GDP 投资项，其折旧又在云算力公司营收里再次显现

考虑乘数加重复，调整后 γ 约 0.02-0.04。

#### 路径二：从已采纳 AI 的企业实证倒推（γ ≈ 0.05-0.08）

Brynjolfsson 等学者在 2024-2025 年用美国劳工统计局（BLS）企业层面调研数据做的研究估算："已采纳生成式 AI 的企业，员工生产率提升 5-15%"。2025 年美国企业 AI 采纳率 8-12%。

按生产率提升 7% 中位数、采纳率 10% 算，宏观加权平均生产率提升 0.7%。假设这 0.7% 完全归因于 C（不归 K 也不归 L），按 5 年累积 = 3.5% 总产出贡献，反推 γ：

3.5% ≈ γ × Δlog(C)；Δlog(C) 取 2020-2025 算力存量年化增长 80% 累积 5 年 = log(18.9) ≈ 2.94。

γ ≈ 0.012 / 年 × 5 年累积 ≈ 0.06。

#### 路径三：类比 IT 资本的早期生产率渠道（γ ≈ 0.12-0.18）

Brynjolfsson 的 J 曲线研究（《The Productivity J-Curve》2018 加后续）发现 IT 资本对 TFP 贡献存在 10-15 年的延迟实现期——IT 投资进生产函数后，TFP 提升要等到企业流程、组织结构、人力资本调整完成才显化。

1990-2007 这段美国 IT 黄金期，IT 资本对 TFP 增长贡献年化约 0.8-1.2 个百分点。

把算力看作 IT 资本的延伸加速，按 IT 资本份额 α_IT 约 0.10-0.15、算力作为 IT 资本子集占比 50%+ 算，C 的弹性 γ 约 0.05-0.10。

如果算力是 GPT 通用目的技术（详见 §6），乘数效应更大，γ 可以推到 0.12-0.18。

| 路径 | γ 估算区间 | 关键假设 | 限制 |
|---|---|---|---|
| 路径一（要素份额倒推）| 0.02-0.04 | 算力支出占 GDP ~1.5%，按完美竞争 / 边际产品定价 | 算力存量增速高，乘数效应未充分计入 |
| 路径二（采纳企业实证倒推）| 0.05-0.08 | AI 采纳率 10% × 生产率提升 7% 累积 5 年 | 采纳率数据宽，5 年累积有重复计入 |
| 路径三（IT 资本类比）| 0.05-0.18 | 算力 = IT 资本延伸 + GPT 通用目的技术 | J 曲线延迟实现期假设强，γ 区间过宽 |

> 来源：三套估算属作者推演，输入数据来自 BLS / Census 2025 企业采纳率、Brynjolfsson, Li, Raymond 2023《Generative AI at Work》NBER WP 31161（路径二推演基础）+ 后续系列论文、Jorgenson-Ho-Stiroh 2008、各家 FY25 10-K 综合资本支出数字。**学术界对 γ 没有公认估算**——本表是按现有方法论倒推一遍。任何报道里出现「算力对 GDP 贡献 X%」这类数字时，都默指了某一套 γ 假设，读时先问那套假设是哪一条。

把三套放在一起看，γ 的合理区间是 **0.05-0.10**，约为劳动份额的 1/10、资本份额的 1/4。

### 2.3 Solow 残差 + TFP 渠道的方法论

把算力作为单独要素 C 写进生产函数，等价于把过去归在 A（残差）里的一部分技术贡献拆出来。这种拆解在经济史上有先例：1960 年代 Jorgenson 把教育从 L 里拆出来作为人力资本（Human Capital），1980 年代 Romer 把研发从 A 里拆出来作为知识资本（Knowledge Capital），1990 年代 Stiroh 等学者把 IT 资本从 K 里拆成独立子要素。每次拆解都让 Solow 残差缩小一个量级——A 不是未知魔法，是还没拆出来的子要素之和。

美国私营非农部门 TFP（按 BLS 口径，Total Factor Productivity）历史增长数据：

| 时期 | TFP 年化增长 | 主要解释 |
|---|---|---|
| 1948-1973 | 1.5-2.0% | 战后高速增长期——制造业 + 基础设施 + 学习曲线 |
| 1973-1995 | 0.3-0.5% | 第一次能源危机 + 滞涨 + 产业结构调整 |
| 1995-2004 | 1.5-2.0% | IT 资本爆发 + 互联网 + 企业流程数字化 |
| 2004-2018 | 0.5-0.7% | IT 红利消化 + 大金融危机 + 长期停滞 |
| 2019-2024 | 1.0-1.3% | 疫情后短暂反弹 + AI 早期渠道 |
| 2025（最新可得年度）| 0.8% | BLS 2026-03 公告（2025 年度数据，BLS 2026-03 发布；区分 2023 年度 0.8% 数据于 BLS 2024-03 发布）|

> 来源：BLS Productivity News Releases（bls.gov/news.release/prod3.htm）2024-2026 + Jorgenson-Ho-Stiroh 2008 + CBO TFP Growth in Historical Perspective 2013 + 1948-1973 子区间数据来自 BLS 历史数据库（按 SIC 转 NAICS 口径）。**子区间端点选择影响均值**——不同研究取不同断点（如 1995 vs 1996，2004 vs 2007）。表中区间是综合主流文献。

读这张表注意三件事。

**第一**，TFP 增速在 1995-2004 出现明显反弹，被学术界广泛归因为 IT 资本对生产函数的贡献。这段反弹有 8-10 年滞后于美国大企业 IT 投资 1985-1995 的爆发期。

这就是 Brynjolfsson 的 J 曲线现象：IT 投资到企业流程、组织结构、人力资本调整再到生产率显化，三阶段累计 10-15 年。

AI 周期始于 2022 年底 ChatGPT 发布，算力资本支出拐点在 2024。按 J 曲线类比，AI 对 TFP 的可观测贡献要等到 2032-2037 才会在 BLS 月度数据里清晰显化。

**第二**，2025 年度 TFP 增速 0.8%（BLS 2026-03 发布）落在历史中位数下方，与 2019-2024 的 1.0-1.3% 接近。还看不到 AI 加速 TFP 的早期信号。这个结果有两种解释：

- AI 仍在 J 曲线的投资期，生产率提升还没显化
- AI 对生产率的实际贡献小于市场预期（Acemoglu 2024-05 NBER paper 的观点）

本书不站队，把两种可能性都摆出来。

**第三**，跨章节读法。任何 2026 年说 AI 已对美国 TFP 贡献 X 个百分点的研究都过早。按 J 曲线时滞，2026 年的 TFP 数据反映的还是 1980-2015 年的 IT 与数字化资本积累，AI 算力的贡献远没到显化期。这件事在第 25 章会用 BLS 微观数据加 Brynjolfsson 等学者的企业层实验数据展开。

算力作为生产要素的合理位置在于沿 Jorgenson / Romer / Stiroh 的传统把 C 从 A 里拆出来，而非破坏宏观经济学框架。拆解的具体渠道（自动化 / 增强 / 位移）后面有专章展开。

γ 的合理区间 0.05-0.10、TFP 渠道的方法论传统、5-15 年 J 曲线延迟实现期，这三件事是后续讨论算力对 TFP 贡献的对照基准。

### 2.4 γ 估算的现实困难

要承认一个不舒适的事实：γ 的估算在学术界没有公认值。本书后续涉及算力对 TFP 贡献的讨论都是在区间里讨论，不会给点估计。

学术界对 γ 难产的根源在三层结构性困难。

**困难一：算力存量的有效价值测算缺乏标准方法**。

传统资本 K 用账面固定资产净值。但算力的账面值跟使能性不一一对应——一张闲置的 H100 跟一张 80% 利用率的 H100 提供的经济价值差 4-5 倍。

要测 K_compute 的有效存量，要把利用率、软件栈成熟度、集群完整性、跨代技术含量这四个变量综合起来。目前没有标准化的测算方法。BLS 在 2025 年开始推进 AI 资本测算项目，但完整方法论至少要到 2027-2028 才能稳定。

**困难二：微观证据有限**。

要从企业层面实证 γ，需要企业 AI 投入到企业产出 / 利润 / 生产率的微观数据。这种数据目前只有 Brynjolfsson 等学者在客服、编码、法律研究等少数行业做了实验，跨行业加跨规模的大样本数据要等到 2027-2030 才会有。

**困难三：内生性问题**。

采纳 AI 这个变量与企业自身的技术能力、管理水平、组织灵活性高度相关——能跑实验测出 AI 采纳 → 生产率提升 X%的企业本身就是高生产率企业。

把这个内生性剔除掉后，AI 对 TFP 的真实贡献可能远低于 AI 采纳企业对未采纳企业的简单对比。Acemoglu 2024-05 paper 主要批评的就是这点。

三层困难压在一起，γ 在 0.05-0.10 这个区间的合理性只是按现有方法论倒推的最佳估计，不是实证测出来的稳定数字。本书后续涉及 γ 时一律标业内推演，不在缺口处给点估计。

### 2.5 与传统资本 K 的边界划分

还要面对一个细节问题：算力是 K 的子集（K_compute ⊂ K），还是独立要素 C（与 K 平行）？两种处理对生产函数的形式与估计结果都有影响。

按算力 ⊂ K 处理：算力归入资本总量 K，按其份额贡献于 K^α 项。优势是与传统 Solow 增长会计兼容，不用重写框架。缺点是把 GPU 与厂房、机器、卡车放在同一桶里——折旧速度、跨代技术含量、产出弹性都极端异质，K^α 这一项的平均行为被算力扭曲。

按算力 ∥ K 处理：算力作为独立要素 C 进 C^γ 项，与 K^α 并列。优势是算力的高折旧率、跨代技术含量、与 IT 资本类似的使能性被显式刻画。缺点是数据要求高——需要独立测算算力存量的有效价值（不只是账面固定资产），还要解决算力使用量与算力存量的关系（自建、租赁、API 调用的口径打通）。

经济学家在 1990-2010 年代处理 IT 资本时走的是 ⊂ K 但单独测算其份额。Jorgenson-Ho-Stiroh 2008 估算 IT 资本对 1995-2007 美国 TFP 年化贡献 0.8-1.2 个百分点。

本书倾向类似处理：算力是 K 的子集，但按其份额、折旧率、技术弹性单独测算其对 TFP 的贡献渠道。具体到生产函数形式：

```
Y = A · K_other^α₁ · K_compute^α₂ · L^β
其中 α₁ + α₂ + β = 1，K = K_other + K_compute
```

α₂ 就是前文 §2.2 讨论的 γ（即 K_compute 的产出弹性），合理区间 0.05-0.10。

## 3. 三重身份之二：作为资产的折旧与残值

这张表里有几件事被市场反复忽视。

**第一**，超大规模云厂在 2022-2023 年集中把折旧期从 4 年拉到 6 年，比 AI 资本支出大潮（2024 年）早 1-2 年。

从时间顺序上看，折旧期延长在前、AI 资本支出加速在后，两件事先后发生而非同时——超大规模云厂先把会计屋子打理好，再放手砸资本支出。这件事的内生性（财务激励是资本支出加速的诱因之一）后续第 17 章会展开。

**第二**，亚马逊在 2024-2025 做了反向操作：FY24 把服务器从 5 拉到 6 年，FY25 又从 6 缩回 5 年（部分资产）。

亚马逊官方理由是 AI 训练加速导致设备实际寿命缩短。亚马逊 CFO Brian Olsavsky 在 Q4 2024 财报电话会上做了相应表述。

这条反例打破了所有超大规模云厂都在拉长折旧期的简化叙事。超大规模云厂之间对 GPU 真实寿命的判断已经出现分化。这件事在第 17 章会被作为关键反例展开。

**第三**，CoreWeave 6 年、Nebius 4 年、Lambda 5 年——GPU 云行业的折旧期分化达 2 年。

同行业不同公司用不同折旧期，会让 Adjusted EBITDA margin 在公司之间不可比。CoreWeave 2025 Q3 Adjusted EBITDA margin 61%，Nebius 同期 32.6%，其中约 20 pp 差距源于折旧期会计选择。

**第四**，Burry 在 2025-11 公开主张超大规模云厂 GPU 真实折旧应为 2-3 年。

论据有二：H100 二手价 2024-2026 跌 70%+；Blackwell 上市挤压 Hopper 训练用途。

如果 Burry 的反情景成立，超大规模云厂当前的会计利润（尤其是 Meta、微软、Oracle）会被打折 30-50%。这件事的完整压力测试在第 16 / 17 / 31 章。

### 3.2 折旧期 → 单卡经济的杠杆

把6 年 vs 4 年 vs 3 年这件抽象的会计选择翻译成单卡经济。

| 折旧期假设 | 单卡 H100 整机 \$32,000 | 年折旧 (USD) | 月折旧 (USD) | 单 GPU-hour 折旧成本（按 85% 利用率） |
|---|---|---|---|---|
| 6 年直线（CoreWeave / MSFT / GOOG）| 残值 12.5% (\$4,000) | \$4,667 | \$389 | \$0.62 |
| 5 年直线（Meta 至 FY24 / AMZN 现行）| 残值 10% (\$3,200) | \$5,760 | \$480 | \$0.77 |
| 5.5 年直线（Meta FY25 现行）| 残值 11% (\$3,500) | \$5,182 | \$432 | \$0.69 |
| 4 年直线（Nebius / 超大规模云厂 2022 前）| 残值 0% | \$8,000 | \$667 | \$1.07 |
| 3 年直线（Burry 反情景）| 残值 0% | \$10,667 | \$889 | \$1.43 |

> 假设：\$32,000 是 H100 SXM5 整机分摊到单卡的采购成本（含服务器 + 网络 + 机架），按 SemiAnalysis + Silicon Analysts 2025 综合 teardown 数据。残值假设按公司方常用口径，CoreWeave / Nebius 不单独披露。

按 H100 长合约价 \$2.35/h（Silicon Data 2026-03）+ 运营成本 ~\$0.45/h（电 + 冷却 + 人员 + 网络分摊），单卡毛利 \$2.35 - \$0.45 - 折旧 = ?

- **6 年折旧**：\$2.35 - \$0.45 - \$0.62 = **\$1.28/h 单卡毛利**
- **5 年折旧**：\$2.35 - \$0.45 - \$0.77 = \$1.13/h
- **4 年折旧**：\$2.35 - \$0.45 - \$1.07 = \$0.83/h
- **3 年折旧（Burry 反情景）**：\$2.35 - \$0.45 - \$1.43 = **\$0.47/h**（比 6 年情况掉 63%）

**折旧期 6 年对 3 年这一个会计选择，单卡毛利相差 2.7 倍**。这是超大规模云厂与 GPU 云财务报表调皮性的核心来源——账面利润对一个会计变量极端敏感。

### 3.3 融资租赁：隐性资本支出放大

折旧政策只是会计杠杆的一面，另一面是融资租赁（finance lease）。

按美国会计准则（ASC 842），公司租赁资产分两类：

- **经营租赁（operating lease）**：租约风险归出租人，承租人在表内确认使用权资产加租赁负债，但不计入资本支出，租金按期计入运营支出。
- **融资租赁（finance lease）**：租约风险归承租人（实质上拥有资产），承租人确认使用权资产加租赁负债，资产按使用寿命折旧、利息按期计入。会计上等同于借钱买资产，表内不进资本支出，但实质是资本支出。

微软在 2024-10-01 CNBC 报道里披露 FY24 一年内累计签约但尚未生效的融资租赁超过 \$100B。这部分尚未生效的融资租赁在传统资本支出数字（\$64.6B FY25）之外，相当于把微软真实 AI 基础设施投入低报 \$100B+ 量级。

微软财报上 \$64.6B 资本支出并不等于 \$64.6B AI 基础设施投入。按可推算口径：

- MSFT FY25 实际资本支出 \$64.6B
- 加当年新生效融资租赁折算约 \$50B（假设 \$100B+ 未生效合同按 5 年均摊加部分当年生效）
- 合计约 \$115B 量级
- 如计入全部未生效融资租赁现金价值则可达 \$150B+

精确测算见第 17 章。同样的隐性放大也出现在 Oracle、Meta、Google，规模不及微软。

融资租赁的会计选择对资产负债表读法有三重影响：

**一**，总负债被低估（如果只看长期债务那一行）。

融资租赁负债单独列示但不计入有息债务科目。读超大规模云厂净负债时要把融资租赁负债加进去。微软 FY25 总融资租赁负债（在表上和尚未生效合计）业内估算 \$200-260B 量级，约为传统长期债务 \$40B 的 5-6 倍。

**二**，自由现金流被高估。

融资租赁的购置部分不进资本支出（不扣自由现金流 FCF），但利息部分进运营支出（扣 FCF）。FCF 数字的真实算力可支配现金比账面值低 10-20%。

**三**，跨公司可比性被破坏。

微软重度用融资租赁锁定建成的数据中心容量，Meta 主要用自建（资本支出真实进表），亚马逊介于两者之间。哪家超大规模云厂 AI 投入最大这件事如果只看资本支出数字，微软会被严重低估。

这条伏笔在后续算力商业模式与超大规模云厂账本两章展开，会把融资租赁隐性放大拆到具体公司具体季度的粒度。

### 3.4 残值假设的产业证据

折旧期长，要靠残值假设支撑。CoreWeave 的 6 年折旧等于假设 H100 / H200 / B200 在 6 年后仍有 10-15% 残值。这件事的产业证据走向支持与反对两条相反路径。

#### 支持长折旧期

- H100 长合约价 2025-10 低点反弹到 2026-03 \$2.35/h。反弹源于 Blackwell B200 / B300 在 2025 H2 到 2026 H1 量产爬坡比预期慢，H100 在推理工作负载上需求被支撑。
- theCUBE Research 2025 年的 GPU Value Cascade 框架：第 1-2 年用于前沿训练、第 3-4 年用于高端推理、第 5-6 年用于批处理推理加分析。GPU 经济寿命远长于前沿训练寿命。
- 部分老 GPU 在不同地区有二次寿命。Hopper、Ampere 卡在新兴市场、主权 AI 项目（第 23 章）仍有需求，残值不会归零。

#### 反对长折旧期（Burry 反情景）

- H100 二手价 2023 Q4 \$40,000+，到 2026 Q1 \$6,000-9,000，3 年累计跌 70%+。
- 同价位点上 B200 训练性能 2.5 倍 H100，让 H100 在前沿训练里被边缘化。
- Anthropic、OpenAI 等模型公司在 H200 上市后向更多 H200 加 B200 倾斜，H100 进入次级训练加推理段。
- 推理工作负载本身也在跨代加速进化。B200 在推理上算力 / W 比 H100 高 2-3 倍，让 H100 在推理端的竞争力也随时间衰减。

两条路径的差额是超大规模云厂 5-6 年折旧期是否合理的核心争议，也是 Burry 与超大规模云厂 CFO 之间的会计辩论焦点。

### 3.5 算力作为资产的三个非典型属性

把算力放在传统资本品（厂房、机器、运输工具）的对照下，有三个属性让它显得非典型。这些非典型属性是超大规模云厂财务读图困难的根源。

**第一**，残值曲折下行而非平滑递减。

传统机器残值大致随使用年限平滑下降：卡车第 1 年残值 80%、第 5 年 40%、第 10 年 15%。

GPU 残值不是平滑曲线。上一代旗舰（H100）在新一代旗舰（B200）上市前是稀缺品，残值在 100%+ 区间（一卡难求）；新一代上市后 6-12 月内残值断崖式下跌 50%+。这是代际拐点，不是线性折旧。

把直线折旧法套到 GPU 上本身就有方法论争议。但 GAAP 允许的加速折旧法应用范围有限，超大规模云厂实践上仍按直线折旧。

**第二**，资产价值依赖软件。

一台 H100 在没有 CUDA / TensorRT / vLLM 的环境下几乎没有价值——硬件价值依赖软件栈支持。

英伟达在 2024-2025 通过软件栈升级让 H100 的推理吞吐量涨 2-3 倍，让 H100 残值的现金流支撑高于纯硬件折旧所示。

但这种软件价值不进资产负债表，会计上仍按物理硬件折旧。这是算力资产价值与会计价值之间的结构性张力。

**第三**，全行业同向折旧暴露。

传统资本品在不同行业有不同折旧节奏（卡车物流、厂房制造、计算机办公），单一行业事件不会触发全市场资产同步重估。

算力不同：超大规模云厂、GPU 云、模型公司在同一代际节奏下购置同一批 GPU。任何关于 GPU 真实寿命的新证据都会同步冲击 5 家以上上市公司的折旧期假设。

Burry 2025-11 公开主张 GPU 真实寿命 2-3 年时，市场对 Meta、微软、Oracle、CoreWeave 的会计争议同步出现。这是单一资产类别的系统性风险。

三件事压在一起，算力作为资产在会计与经济两层之间存在持续紧张关系。会计层面超大规模云厂用 5-6 年直线折旧、残值 10-15%；经济层面 GPU 真实寿命周期、软件依赖、全行业同步暴露让账面价值与经济价值之间的差距可能扩大。算力作为资产不能套传统会计直觉。

### 3.6 资本支出强度与折旧政策的内生关系

把折旧政策与资本支出强度放在一起看，会发现一条非直观的内生关系：超大规模云厂在资本支出强度上升前先调整折旧政策——会计屋子先打理好，再放手砸钱。

时间序列对照：

- 微软：2022-08 公告 4 改 6 年折旧；2023-2026 AI 资本支出从约 \$30B / 年涨到约 \$80B / 年
- Google：2023-01 公告 4 改 6 年折旧；2023-2026 AI 资本支出从约 \$30B / 年涨到约 \$75B / 年
- Meta：2022 Q2、Q4 两次延长，2025-01 进一步延长到 5.5 年；2024-2026 AI 资本支出从约 \$30B / 年涨到约 \$72B / 年

折旧期延长在资本支出加速 1-2 年前完成。这不是巧合。折旧政策决定了同样资本支出在利润表上扣多少。把 4 年折旧延长到 6 年，相当于把单年折旧摊销减少 33%，当年利润空间增加。利润空间增加，资本市场对资本支出加速的容忍度变高，公司管理层有空间继续扩张资本支出。

这条内生关系的隐含含义是：超大规模云厂财报上 AI 投入的数字与 AI 实际投入之间的差距不仅来自融资租赁（隐性资本支出放大），还来自折旧政策选择（隐性利润放大）。

真正的 AI 投入规模要把折旧政策、融资租赁、传统资本支出三件事一起读。单独读资本支出数字会严重低估。

对投资者读者而言，这条内生关系还有一个直接的衍生判断：当一家超大规模云厂突然反向调短折旧期时，是一个值得警惕的信号。

亚马逊 2025-02 把服务器从 6 年缩到 5 年（部分资产），明面理由是"AI 加速导致设备实际寿命缩短"。内涵可能是公司方对 GPU 真实寿命的内部判断变得保守，保守判断会传染到其他超大规模云厂的折旧政策审查上。

亚马逊这一动作之后，Burry 在 2025-11 公开主张更激进的 2-3 年折旧期。市场对超大规模云厂折旧期假设的共识区间在 2025 年下半年开始变得不稳定。这条争议在 2026-2027 是否会进一步扩散，是第 31 章（12 个争议）会单独讨论的开放问题。

## 4. 三重身份之三：作为商品的 token / GPU-hour 现货市场

把同一张 H100 再换一个视角——它产出的算力服务在现货市场上按什么定价？

这个市场跟传统大宗商品（原油、铜、铁矿石）有类比，但也有结构性差异。

### 4.1 从大宗商品类比谈起

把算力作为商品的最干净类比，是大宗商品中的电力——同样按瞬时供给与实时消费、不易存储、价格小时级波动。

"Compute is the new electricity"（算力是新电力）这句话在 2024-2025 年被英伟达 CEO Jensen Huang、微软 CEO Satya Nadella 反复使用，本质是把算力的商品属性向电力对齐。

这个类比只对一半。

算力跟电力的相同之处有四件：

- 都是几乎所有下游产业的生产投入
- 都不易存储——GPU 不工作时残值不归零，但单位时间的算力流如果不消耗就完全消失
- 都按小时 / 兆瓦时的瞬时单位计价
- 都受电网、数据中心、调度系统的物理约束

算力跟电力的不同之处更关键：

- 电力同质性高，1 度电跟 1 度电完全可互换；算力跟具体硬件代际、软件栈、集群拓扑深度耦合
- 电力市场有半个世纪的监管加标准化期货市场；算力市场刚开始有 Silicon Data 这样的指数
- 电力供给弹性受发电资产、输电网、容量市场约束；算力供给弹性受芯片产能、HBM 产能、数据中心电力接入约束（详见第 10 / 11 章）

把算力定位为还没完全标准化的电力是有限但有用的类比。这个定位让电力市场学到的容量定价、实时市场、长期 PPA、期货对冲那套工具，可以被借鉴来思考算力市场未来 5-10 年的演化方向。

这条曲线的形态像极了大宗商品周期——2023 Q4 高峰、2024 全年跌、2025 Q4 低点、2026 Q1 反弹。H100 跟原油不同的地方有三件：

**第一**，供给极度刚性。原油的供给响应周期是 6-18 个月（钻井到出油），H100 是 12-18 个月（台积电 CoWoS 月产能爬升加 SK 海力士 HBM3e 量产）。算力供给的硬刚性让价格波动比原油更激进。

**第二**，需求高度异质。原油在炼厂出来后跟谁烧没区别。H100 跟在哪个集群有强差异——NVLink 拓扑、数据中心位置、客户类型。\$2.35/h 长合约价跟 \$3.5/h 现货价之间是不同产品，不是单纯折扣。

**第三**，价格透明度低。原油有 ICE / NYMEX 期货市场，价格小时级公开。H100 没有标准化期货市场，价格通过 Silicon Data、SF Compute、私下双边谈判形成，市场参与者拿到的价格经常滞后 1-2 个月。

这件事的结构含义是：算力作为商品在价格发现层面比传统大宗商品弱。市场对 H100 真实供需的认知滞后于物理供给。这是第 12 章（2027 拐点）的一个关键判断依据。

- 硬件代际进步：H100 到 B200 算力 2.5 倍
- 推理软件栈优化：vLLM、TensorRT-LLM、连续批处理（continuous batching）给 token 吞吐量 2-5 倍
- 模型架构进步：MoE、MLA、蒸馏给 token 吞吐量 2-10 倍

三个杠杆在 2024-2026 同时起作用，导致 per-token 价跌幅远超过单纯硬件迭代。

### 4.4 一个商品市场，两个分裂的客户群

GPU-hour 与 token 看起来是同一供给端（GPU 农场）的两种产品形态，客户群完全分裂。

**GPU-hour 客户**：

- 自建模型公司（OpenAI、Anthropic、xAI）——长合约 5-7 年照付不议
- 大企业自训（金融、制药、自驾）——短期项目租赁，3-12 月
- 学术与政府研究——通常通过超大规模云厂的政府云
- 中型 AI 创业公司——按月或季度签合约

**token 客户**：

- 应用开发者（Cursor、Perplexity、微信小程序）——按调用量
- 终端消费者（ChatGPT 订阅）——按月固定费
- 企业部署（客服、内部 RAG、自动化办公）——按使用量加包年合约

两种客户对算力价格的敏感度差一个数量级。token 客户主要看每百万 token 单价、延迟、上下文窗口；GPU-hour 客户主要看集群完整性、InfiniBand 拓扑、长合约价。同一张 H100 在两个市场上分别报价，几乎不交叉。

这件事的产业含义：超大规模云厂与 GPU 云面对 GPU-hour 市场（B2B 长合约加短期租赁）；模型公司加应用层公司面对 token 市场（B2C 与 B2B SaaS）。两层之间的价值流由第 15-18 章完整拆解。

### 4.5 与传统大宗商品的三个关键差异

把算力作为商品的形态与传统大宗商品（原油、铜、铁矿石、电力）做横截面对照，有三个关键差异决定了 AI 算力市场的运行规律不能套传统商品市场的直觉。

#### 差异一：算力没有现货交易所

大宗商品都有标准化期货市场，价格小时级公开、可对冲——原油有 ICE / NYMEX，铜有 LME / SHFE，铁矿石有 SGX / DCE 普氏指数。算力没有。

Silicon Data H100 Rental Index 是目前最接近 H100 价格指数的数据源，但它只覆盖部分玩家，发布频率周度或月度，且不可对冲。市场参与者要做空 H100 算力价格，只能通过股权（做空 NVDA 或 CoreWeave）或者高度间接的衍生品组合，没有直接的算力期货。

这件事的产业含义有三层：

- 价格发现滞后于物理供需变动 1-2 个月，市场对真实供需的认知有时间差
- 超大规模云厂与 GPU 云在签长合约时实际承担了价格走势风险，不能像航空公司对冲油价那样对冲算力价格
- 算力作为生产投入对其他产业（自动驾驶、制药、金融）来说，没有可参考的远期价格曲线做 5-10 年规划

#### 差异二：商品口径的非标准化

原油按 API 度（密度）加含硫量分等级（WTI / Brent / Urals），同一等级在全球范围内可互换。

H100 按代际、形态、集群完整性、软件栈分级。8 张 H100 SXM5 加 NVLink Switch 加 InfiniBand 400Gb/s 加 CUDA 12 加 TensorRT-LLM，不等于 8 张 H100 PCIe 加没有 NVLink 加弱网络。同一张 H100 GPU 在两种部署形态下的算力商品价值差 30-60%。

这导致 H100 价格指数本身的粒度问题。Silicon Data 公开的 \$2.35/h 长合约价是综合多种部署形态的平均，实际可比性弱。要做严肃的算力价格分析，必须按部署形态分级单独定价——但市场上没有这样的分级指数。

#### 差异三：供给响应时间与需求弹性的不对称

原油供给响应是 6-18 个月（钻井到出油再到炼厂），需求弹性约 -0.1 到 -0.4（短期低、长期升）。

算力供给响应是 12-24 个月（台积电 CoWoS 月产能爬升加 SK 海力士 HBM 量产加服务器组装加数据中心电力接入），算力需求弹性按 §5 的 Jevons 反弹效应是正的——单价下降反向激发更多需求。

供给慢、需求弹性又是正的，两个条件叠加造成价格暴涨与暴跌、供给跟不上又跟不下的振荡。算力周期的硬刚性叠加软弹性，决定了 2023-2026 这一轮振荡——H100 现货 2023 Q4 高峰 \$10/h，2025 Q4 低点 \$1.7/h，2026 Q1 反弹 \$2.35/h。

6 倍区间的振荡幅度，在传统大宗商品里是历史顶部级别的剧烈程度（原油 2014-2016 跌 75% 已被认为是大型周期）。

把三件事压在一起，算力作为商品市场的运行规律有自己的逻辑——既不能套软件 SaaS 的边际成本接近零直觉，也不能套传统大宗商品的标准化期货直觉。后续 2027 拐点、杰文斯三段论、GPU 云财务几章会反复回到这条市场逻辑。

### 4.6 算力商品市场的反身性裂缝

把算力作为商品的论述收尾在一件 Soros 反身性意味很重的事上：算力价格、客户合同、客户信用、算力供给融资再回到算力价格的循环回路。

CoreWeave Q3 2025 8-K 披露合同储备（Revenue Backlog） \$55.6B，其中 OpenAI 累计 \$22.4B / 5 年，微软长约市场估 \$10B+。这部分合同储备在会计上是已签合同未交付的未来营收，CoreWeave 用它做了两件事：

- 给上市估值故事提供未来 5 年现金流可见性
- 作为债务融资的抵押与信用支撑——2026-02 用 Meta 合同抵押拿到 \$8.5B 融资

这条链条的反身性走法：算力商品价格上涨 → 客户的算力刚需叙事强化 → 长合约规模扩大 → CoreWeave 与同类公司能融到更多债 → 买更多 GPU 扩供给 → 算力市场总供给上升 → 价格压力增大。

如果算力刚需在某代模型上松动（更高效算法或推理需求增速放缓），整条链条会反向走：长合约新增量下降 → 融资难度上升 → 已购 GPU 转入二级市场 → 现货价加速下跌。

这条反身性裂缝是 2024-2026 美国 GPU 云行业的核心系统性风险。把算力作为商品讨论时，不能只看现货价加长合约价两个静态数字，要看价格 → 信用 → 融资 → 供给 → 价格这条动态环。

## 5. Jevons 反弹效应：算力价格暴跌之后

19 世纪英国经济学家 William Stanley Jevons 在 1865 年的《The Coal Question》里提出一个反直觉观察：当蒸汽机效率提升、煤炭使用效率上升时，煤炭总消费量不下降，反而上升。

原因是效率提升压低了使用成本，激发出更多新用途（铁路、纺织、钢铁），新需求超过效率节省。这就是 Jevons paradox（杰文斯悖论），也叫反弹效应。

把这件事翻译到算力上的早期证据，DeepSeek-R1 是一个干净的简化案例。

### DeepSeek-R1 实验

2025-01-20，中国 AI 实验室 DeepSeek 发布 R1 模型。推理性能与同期 OpenAI o1 接近，API 价格比 OpenAI 同档低 20-50 倍。

R1 首发单价 \$0.07 / \$2.19 per 1M tokens（输入 / 输出），同期 OpenAI o1 是 \$15 / \$60 per 1M tokens。输入价差 214 倍按 R1 首发价对 OpenAI o1 同期输入价折算；按 2026-05 R1 当前价（缓存未命中输入 \$0.55）折算约 27 倍。无论按哪个口径，价差都是数量级，叙事不变。

预期反应：DeepSeek 拿走部分 OpenAI 市场份额。

实际反应有四件：

- 英伟达单日市值蒸发 \$589B（2025-01-27 收盘）。市场判断算力效率提升 → 算力总需求下降，对 NVDA / AVGO / TSM 集体重估。
- DeepSeek R1 在第三方平台的使用量在 1 个月内涨 20 倍。
- OpenAI、Anthropic、Google 同期没有用户大规模流失。付费 token 总量在 2025 Q1-Q2 反向上涨。
- 全球推理 token 调用总量 2025 全年估计涨 50 倍量级。

英伟达在 2025-02-26 FY25Q4 财报上，数据中心营收同比 +93%。市场用半年时间把 DeepSeek 让算力需求下降的初判反转为 DeepSeek 让算力需求加速——典型 Jevons 反弹效应。

这件事的机制可以一句话总结：算力单价跌 99%、使用量涨 5,000%，算力总支出涨 50 倍。

Jevons 在算力上的早期证据从 DeepSeek-R1 之后已经显现。完整机理（前训练 / 后训练 / 推理三段算力的弹性差异、agent AI 的几何级数倍增、推理范式从单次查询到多轮推理）需要在第 13 章单独展开。

### Jevons 在算力上的边界条件

Jevons 反弹效应在算力上有几个边界条件。

**条件一：使用门槛低、应用空间未饱和**。

Jevons 在 19 世纪英国煤上成立，是因为蒸汽机降低了煤的单位有效热值成本，而蒸汽机本身可以驱动几乎所有工业流程（纺织、铁路、采矿、钢铁），使用空间几乎无限。

算力在 2024-2026 也满足这个条件——LLM 单 token 价格下降激发的应用空间（聊天机器人、编码、内容生成、agent、RAG、客服）远没达到饱和。

如果未来某代 AI 算力价格趋近 0，但 AI 已经覆盖一切应用（边际应用价值低），Jevons 反弹会减弱。Jevons 反弹在某一代 AI 上终会减弱，只是哪一代不确定。

**条件二：算力使用的瓶颈不在算力本身**。

如果算力下游的瓶颈（电力、数据中心、人才、监管）让算力使用受限，Jevons 反弹会被瓶颈封顶。

2024-2026 美国 PJM 容量市场清算价两年跳涨 11 倍，电力已经成为算力下游的硬约束。算力便宜了 100 倍，但电不够如果成为现实，Jevons 反弹会卡在电力天花板上。第 10 / 14 章会展开。

**条件三：替代关系还是互补关系**。

Jevons 在煤上的传导是煤效率提升带动煤需求上升，煤与蒸汽机互补。如果某种新技术让 AI 的算力依赖下降（更高效的算法、更小的模型、神经形态芯片），那时算力会进入被替代轨道。

2024-2026 的实证仍偏向算法效率提升 → 模型变大 → 算力需求更大，互补关系占主导。跨代周期的算法或硬件结构性突破是第 31 / 32 章要讨论的尾部场景。

### 市场对需求弹性的反复修正

DeepSeek-R1 之后市场对算力需求弹性的反复修正本身就是个有意思的实验。

2025-01-27 英伟达单日 -\$589B 市值，市场默认算力价格 ↓ → 算力总需求 ↓，套用传统供需直觉。

2025 年下半年英伟达数据中心营收持续创新高，市场修正为算力价格 ↓ → 算力总需求 ↑，Jevons 反弹效应成立。

两次反向修正的时间差只有 6-9 个月。市场对算力作为商品的需求弹性符号都需要时间形成共识。

延伸含义：算力相关股票（NVDA / AVGO / TSM / SK 海力士）的估值波动有相当一部分来自市场对算力需求弹性的认知反复，而非基本面变化。这是第 30 章估值章节会展开的反身性溢价的一个具体例子。

算力价格跌 → 算力总需求暴涨这个反直觉关系，是后续判断 AI 需求弹性的基础。同时要标清楚：Jevons 是一个有条件的经济学规律，不是永远成立的物理定律。完整的杰文斯三段论（前训练 / 后训练 / 推理三段算力的弹性差异）后面有专章单独展开。

### Jevons 在算力上的宏观投资含义

如果 Jevons 在某代 AI 上持续成立——单位算力价格年化跌 50%，总使用量年化涨 100%，总算力市场年化涨 100%——这相当于算力市场是一个价格暴跌但市场规模翻倍的非典型增长行业。

这种行业的赢家是单位成本快速下降加渗透率快速扩张的玩家——英伟达、台积电、SK 海力士这种吞吐量巨大且代际加速的供应商。

输家是单位价格依赖维持的玩家——不能跨代加速吞吐量的 ASIC 玩家、不能用规模分摊固定成本的小型云。

这件事对后面英伟达护城河、中国算力路径两章会有具体含义。

### Jevons 与能力阶梯耦合

Jevons 反弹效应的另一个观察角度是能力阶梯（capability ladder）。

AI 模型从 GPT-3 到 GPT-4 再到 o1 / o3 的能力阶梯每跨一步，能解决的任务类型扩大一个量级——简单文本生成 → 长文推理 → 复杂决策与 agent 任务。

每一步能力升级激发的新需求超过上一步效率提升压缩的需求。这是 Jevons 在算力上能持续 5-10 年的根本机制：只要 AI 能力阶梯还在向上爬，单位算力变便宜激发的新需求总能超过节省。

但能力阶梯不可能永远向上爬。某一代触及能力天花板（如真正的通用决策能力），Jevons 反弹会减弱。

## 6. Acemoglu 任务导向框架与算力替代什么任务

Daron Acemoglu 与 Pascual Restrepo 在 2018-2020 一系列论文里把传统算力替代劳动框架升级为任务导向（task-based）框架。核心观点是：技术替代的对象是具体任务，不是劳动这个抽象类别。

一份白领工作可以拆成 50-100 个具体任务（写代码、开会、写文档、调代码、给客户写邮件等），AI 算力对每个任务的替代率不同。Acemoglu-Restrepo 把任务分三类：

- **位移效应（displacement effect）**：AI 完全替代人执行任务（自动客服），减少劳动需求
- **生产率效应（productivity effect）**：AI 增强人执行任务（Copilot 辅助写代码），增加人均产出
- **重置效应（reinstatement effect）**：AI 创造新任务（prompt 工程师、RAG 维护、模型评估），增加新型劳动需求

算力作为通用任务自动化器在三种机制上同时起作用。具体哪类任务的位移效应大、哪类任务的生产率效应大、新任务的重置规模有多大，是经验问题。

Brynjolfsson 等学者在 2024-2025 用 GitHub Copilot、客服 AI、法律 AI 的微观实验数据测算：使用 AI 的客服员工生产率提升 14%，新员工受益最大（生产率提升 34%）；使用 Copilot 的软件工程师任务完成速度提升 55%。

这些实证已经把任务导向框架具体任务那一层填上了部分数据。

特别要标的一件事：Acemoglu 本人在 2024-2025 多次发文质疑 AI 对 TFP 的真实贡献（如 Acemoglu 2024-05 NBER paper），论点是 AI 的任务替代率在多数白领任务上有限（< 20%），实际 TFP 贡献被市场高估。

这是经济学家内部的争议。Brynjolfsson 偏向乐观，Acemoglu 偏向谨慎。本书不站队，只把双方论据在第 25 / 26 章摊开。

把任务导向框架放在本章三重身份语境里看，有两个对全书理论一致性重要的衍生判断。

**衍生判断一**：算力替代什么任务，决定哪条产业链受冲击。

如果算力主要替代数据处理、客服、简单代码、内容生成这类常规白领任务，受冲击的是业务流程外包（BPO）、媒体编辑、入门级开发岗位。同时激发的是 AI 训练数据标注、prompt 工程师、agent 系统运维、模型评估等新岗位。

任务导向框架不预言 AI 替代所有工作，预言的是 AI 重构任务组合，部分岗位收缩、部分岗位扩张、新岗位出现。

> BPO 即 Business Process Outsourcing 业务流程外包。

**衍生判断二**：算力的产出弹性 γ 不是均匀分布的。

前文 §2.2 给出 γ 区间 0.05-0.10。任务导向框架给的进一步推论是：在某些任务密集型行业（软件、客服、法律研究、简单医学影像分析）γ 可以远高于平均（0.15-0.25），在另一些行业（建筑、制造业的物理操作、高级研究）γ 接近 0。

算力对 TFP 的宏观贡献是各行业贡献的加权平均。单看宏观数字会掩盖行业之间的剧烈分化。

### 6.1 任务导向框架对算力投资的隐含含义

把任务导向框架反过来用，能推出一条对算力产业链投资极端重要的判断：算力的边际产出在不同应用场景差几个数量级。

2024-2026 的产业实证显示，单卡 H100 在不同应用上的美元产出 / 美元投入比：

- **前沿模型训练**（OpenAI、Anthropic、xAI）：单卡产出反映在模型迭代价值上，难以单卡核算。按模型公司的整体经济，单卡 H100 一年的模型价值贡献可能在 \$30,000-100,000，依模型发布节奏与估值口径而变
- **付费 API 服务**（GPT-4o、Claude）：单卡每年推理产出价值 \$15-30K，按 §4 的 token 营收公式估算
- **企业内部应用**（客服 AI、内部 RAG）：单卡每年产出价值 \$5-15K
- **科研与学术研究**：单卡每年产出价值难以以美元衡量，社会回报通常高于商业回报
- **加密货币挖矿**（部分 GPU 老卡的次级用途）：单卡每年产出 \$500-2,000，依币价波动

把这条边际产出谱系放在任务导向框架里看，AI 算力的资源配置效率取决于算力是否流向边际产出最高的应用。

市场机制（超大规模云厂长合约、GPU 云转租、现货市场）部分实现了这种资源配置。但客户集中度极高——NVDA FY26 前 5 大客户占约 40%，第 7 章详谈——让边际产出最高的应用实际上集中在 5-10 家公司手里。

算力作为生产要素的宏观配置效率因此受少数玩家的内部决策影响极大。这件事的反身性含义在第 29 / 30 章单独展开。

### 6.2 算力 / 劳动的替代弹性问题

经济学里讨论技术替代劳动时，关键变量是替代弹性 σ。σ = ∞ 意味着算力完全可以替代劳动；σ = 0 意味着算力与劳动只能互补、不能替代。Cobb-Douglas 生产函数隐含 σ = 1。

实证上，算力与劳动的替代弹性 σ 在不同任务类型上差异极大：

- **数据录入、简单客服、OCR 处理**：σ 接近 ∞，算力几乎可以完全替代人
- **代码编写、内容生成、设计、法律研究**：σ 在 0.5-2.0 之间，算力辅助人但不完全替代
- **复杂决策、物理操作、高级研究、创造性工作**：σ 接近 0.1-0.3，算力是补充工具，不替代决策角色

σ 谱系决定了 AI 对就业的分层冲击：σ 高的岗位先被压缩，σ 低的岗位反而因为算力辅助而生产率提升、薪酬上升。

把 σ 谱系翻译到三重身份语境里：

- **要素视角**：σ 决定了算力作为 K_compute 与 L 之间的边际替代率，影响生产函数的形式（Cobb-Douglas、CES 等）
- **资产视角**：σ 影响企业自动化决策——σ 高的岗位企业愿意把高额算力资本支出砸进自动化，σ 低的岗位企业更倾向算力辅助加增加员工的混合模式
- **商品视角**：σ 高的岗位驱动 token 与 agent 需求爆发，σ 低的岗位驱动 GPU-hour 训练需求（用 σ 低的人才训出新模型，模型再去做 σ 高的任务）

算力 / 劳动的替代弹性是非均匀的，这一点能避免后续讨论 AI 就业冲击时被 AI 替代所有工作或 AI 不替代任何工作两个极端叙事裹挟。

### 7.2 Bresnahan-Trajtenberg 通用目的技术三判据的细节

六个理论里 Bresnahan-Trajtenberg（1995）的通用目的技术框架值得多说一段，因为它直接回答了算力是不是历史级技术这件事。

> 通用目的技术（General Purpose Technology）简称 GPT，与生成式 AI 的 GPT 缩写无关，本节单独标注。

Bresnahan & Trajtenberg 1995 论文《General Purpose Technologies: Engines of Growth?》给出 GPT 三判据：

1. **Pervasiveness（弥散性）**：能在多数产业 / 部门里被使用，不限于单一行业
2. **Continuous improvement（持续改进）**：技术本身在长时间尺度上持续进步，价格 / 性能比持续优化
3. **Innovation complementarities（互补创新）**：在 GPT 之上能激发出大量上游 / 下游 / 邻近产业的互补创新

按这三条判据，历史上被广泛认可的 GPT 包括：蒸汽机（1700s-1850s）、电力（1880s-1930s）、内燃机（1900s-1960s）、ICT 信息通信技术（1970s 至今）。每代 GPT 的扩散周期 30-100 年，对宏观经济的贡献在扩散后 20-30 年才显化，与 Brynjolfsson J 曲线一致。

算力是不是 GPT？按三判据逐条对照：

- **Pervasiveness**：满足。算力已经渗透到金融、制造、医疗、媒体、教育、自动驾驶、客服、法律等几乎所有白领行业，物理产业部分（建筑、农业）也开始有渗透。
- **Continuous improvement**：满足。Epoch AI 数据：前沿训练算力 2010-2024 年化 4-5 倍；per-token 单价 2023-2026 跌 99%+；硬件 FLOPS / W 跨代 2 倍量级提升。三个尺度都满足持续改进。
- **Innovation complementarities**：满足。算力激发的互补创新包括芯片设计（英伟达、AMD、Google TPU、国产 ASIC）、数据中心（CoreWeave、Crusoe）、电力市场重构（PJM 容量市场、SMR 重启）、新型软件栈（CUDA、vLLM、TensorRT-LLM）、AI 应用层（Anthropic、OpenAI、字节豆包），以及众多围绕 AI 的产品类目。

三判据全部满足。算力作为通用目的技术的定位有理论支撑，这是第 24 章的核心结论。

但要标的一个细节是：算力是 GPT 不等于算力一定能驱动宏观 TFP 加速。

蒸汽机和电力分别在 1840s 和 1900s 被识别为 GPT，但宏观 TFP 加速分别要等到 1850s（铁路网建成）和 1920s（电气化工厂普及）。GPT 的扩散到宏观经济显化之间有 20-30 年时滞。

AI 算力在 2024-2026 被确认为 GPT 是合理的，但要看 AI 对宏观 TFP 的决定性贡献，仍要等到 2040-2055 年代。这件事的尾部场景在第 32 章会被作为基准假设之一。

## 小结

第 1 章给了一张物理表（H100 BOM），本章给了一组坐标——五维单位换算、三重身份、六个理论锚点。

### 五维单位换算速查表

FLOP / GPU-hour / token / MW / 资本支出五个量纲不可通约，但可以在三类口径陷阱注释下做有限翻译。这张表是工程师与财务背景读者之间的公共货币，后续 30 章会反复回引。

### 三重身份的三组分析框架

算力作为生产要素 K_compute（进 Cobb-Douglas，γ 业内推演 0.05-0.10，详见第 24-26 章）；作为资产（固定资产加融资租赁，折旧期假设是超大规模云厂利润表最大的杠杆点，详见第 15-17 章）；作为商品（token / GPU-hour 现货市场，Jevons 反弹效应已显化，详见第 12-13 章）。三种身份各自有完全不同的章节归属。

### 六个理论锚点的应用矩阵

Solow / Jevons / Acemoglu / Bresnahan-Trajtenberg / Minsky / Soros，本书理论库的全集。

### 三种身份的分析单位不同

三重身份的分析单位完全不同——这件事后续章节会反复出现：

- 要素视角的单位是 K_compute 在 Cobb-Douglas 里的份额（无量纲）
- 资产视角的单位是 USD（固定资产净值）
- 商品视角的单位是 GPU-hour 或 token

同一个数字在三种视角里的含义不同。以英伟达 FY26 数据中心营收 \$193.7B 为例：

- 要素视角下是全球算力作为生产要素的市场规模代理变量
- 资产视角下是英伟达作为供应方的存量销售总额
- 商品视角下是全球 GPU-hour 现货市场的设备端供给规模

读财报数字时先问自己现在在哪种身份语境里，能避免大部分跨章节的逻辑混乱。
